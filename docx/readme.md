

# 🚀 【AI 架构师交接说明书】veso-phone - 核心 Todo 与 RPG 养成系统架构蓝图

你好，AI 助手。从现在开始，你将作为我的首席前端架构师和全栈开发工程师，协助我开发一款名为 **「veso-phone」** 的纯前端 Web App（PWA）。

这是一个运行在浏览器中、模拟智能手机界面的超级系统。它没有传统后端，所有数据均通过 `Dexie.js` 存储在浏览器的 `IndexedDB` 中。该系统的核心理念是将 **“效率工具 (Todo/番茄钟)”**、**“大模型情感陪伴 (AI 同桌)”** 与 **“RPG 游戏化养成”** 完美融合。

当前阶段，我们需要全力开发该系统的核心模块：**「Todo & RPG 养成 App」**。为了确保你完全理解项目的宏大愿景和底层逻辑，请仔细阅读以下详尽的架构设计、UI 规范以及跨模块打通的底层逻辑。

---

## 壹、 核心技术栈与开发原则

1. **前端框架**：Vue 3 (Composition API, `<script setup>`)。
2. **数据持久化**：`Dexie.js` (封装 IndexedDB)，绝对不使用任何后端数据库。
3. **UI/CSS 规范**：纯 CSS 实现，不依赖重型 UI 组件库。大量使用毛玻璃 (`backdrop-filter`)、极简线框、纸艺质感。**特别注意：在 RPG 面板中禁止使用 Emoji，必须使用纯净的 SVG 或 FontAwesome 图标。**
4. **架构解耦**：严格遵循“视图 (Views) - 逻辑 (Composables) - 数据 (Dexie)”三层分离。

## 贰、 跨模块底层打通逻辑 (极其重要)

这个 Todo App 不是孤立的，它必须与系统内的其他 App（QQ 聊天、记忆库）以及现实世界（物理手机）进行深度数据交互。请深刻理解以下三大打通机制：

### 1. 与「QQ 聊天 App」的打通 (动态 Prompt 注入)
Todo App 产生的所有数据（今日任务进度、RPG 等级、番茄钟专注时长），都必须能被 AI 同桌感知。
*   **实现方式**：在全局的 `usePromptOrder.js`（负责组装发给大模型的 API 消息数组的脚本）中，每次用户在 QQ 界面点击发送消息前，系统会静默读取 `db.todos` 和 `db.userProfile`。
*   **注入逻辑**：将当前的任务状态转化为一段 System Prompt 塞入上下文。例如：*“【系统指令】：用户当前 RPG 等级为 Lv.5。今天计划了 5 个任务，完成了 2 个，还有 3 个未完成。请在接下来的聊天中，根据你的傲娇人设，自然地催促他去完成任务。”*
*   **全局悬浮窗**：在 Todo App 的任何页面，右下角都有一个全局的 AI 悬浮球 (FAB)。点击可从底部弹出一个半屏的 QQ 聊天抽屉，实现边看任务边和 AI 聊天的无缝体验。

### 2. 与「记忆库 (Memory Vault)」的打通
Todo App 中的重要行为，将转化为 AI 的长期记忆。
*   **里程碑写入**：当用户完成了一个困难任务，或连续专注超过 2 小时，Todo 逻辑层会自动向 `db.timeline` (时间轴表) 和 `db.memories` (记忆表) 写入一条记录。
*   **日记素材**：用户在 Todo App 的“每日复盘”中写下总结，如果转发给ai，AI 也会知道，在“AI 起居注/日记”作为素材，写完存入记忆库供用户日后翻阅。

### 3. 与「现实物理手机」的打通 (MacroDroid 联动)
系统需要记录用户在现实中玩手机的时间（如刷小红书 45 分钟），并将其显示在 Todo 的时间轴上，甚至触发 RPG 惩罚。
*   **实现方式 (URL 传参拦截)**：用户手机上的 MacroDroid 会在关闭特定 App 时，自动在后台访问我们的 Web App 链接并携带参数，如 `/?action=log_time&app=小红书&duration=45`。
*   **前端拦截**：在 `App.vue` 的 `onMounted` 中，拦截这些 URL 参数，解析后直接写入 `db.timeline`，随后利用 `history.replaceState` 擦除 URL 参数，做到无感记录。

## 叁、 数据库结构设计 (Dexie.js)

在 `src/db.js` 中，我们需要为 Todo 和 RPG 系统建立或更新以下表结构：

```javascript
db.version(1).stores({
  // 1. 任务主表 (兼容用户任务与 AI 专属任务)
  // owner: 'user' | 'ai'
  // quadrant: 1(重要紧急), 2(重要不紧急), 3(紧急不重要), 4(不重要不紧急)
  // recurrence: 'none', 'daily', 'weekly'
  todos: '++id, owner, title, category, quadrant, status, dueDate, parentId, createdAt, completedAt',
  
  // 2. 番茄钟专注记录表 (用于生成可视化图表)
  pomodoro_records: '++id, taskId, duration, startTime, endTime, status',
  
  // 3. 每日复盘与灵感碎片表
  daily_reviews: '++id, date, content, rating',
  idea_fragments: '++id, content, timestamp, isProcessed',
  
  // 4. 全局时间轴 (混合 Todo 完成记录与 MacroDroid 抓取的 App 使用记录)
  timeline: '++id, type, content, timestamp'
});
```
*(注：用户的 RPG 属性如 `level`, `exp`, `actionPoints`, `bingoGrid` 存在 `userProfile` 的 JSON 对象中，通过 localStorage 或单条 Dexie 记录维护。)*

## 伍、 核心五大 Tab 页面详细设计规范

这是整个 Todo App 的 UI 与交互灵魂，请在编写 Vue 组件时严格遵循以下设计要求：

### Tab 1: 任务票据 (Tab1_List.vue)
*   **视觉风格**：简约、纸艺、平面票据质感。背景采用极浅的米色，带有细微的纸张纹理或虚线分割。
*   **顶部导航**：包含 `< 前一天 | 今天 | 后一天 >` 的日期切换器。
*   **核心交互 (70/30 撕票据布局)**：
    *   放弃传统的上下滑动或左右滑动切换。在同一页面内，左侧 50% 宽度展示 `status: 'todo'` 的待办列表。
    *   右侧 50% 宽度设计为一条“已完成存根 (Done Stub)”。
    *   当用户在左侧勾选完成一个任务时，触发 CSS 过渡动画：该任务条目向右飞入并“贴”在右侧的存根区。另外，也可以加入非todo计划但done了的条目。



### Tab 2: 专注卡片 (Tab2_Focus.vue)
*   **视觉风格**：大卡片流，极简现代风。
*   **功能逻辑**：
    *   页面展示预设的番茄钟模式卡片（如：25分钟快速背词、50分钟深度阅读）。
    *   点击卡片后，弹出面板选择关联的 Todo 任务，以及选择“AI 陪伴态度”（温柔鼓励 / 毒舌监工）。
    *   进入倒计时后，全屏隐藏底栏。屏幕中央是倒计时圆环，上方是 AI 的立绘/头像。
    *   **预生成机制**：开始计时的瞬间，调用 `useTodoAI.js` 预先生成 几 句 AI 的“碎碎念”。在倒计时期间，定时在 AI 头像旁弹出气泡，营造真实的陪伴感，且不消耗额外网络请求。

### Tab 3: 全知视界 (Tab3_Stats.vue)
*   **视觉风格**：数据可视化面板，类似 GitHub 或 Notion 的极简数据风。
*   **核心区块**：
    *   **热力图**：顶部展示类似 GitHub 的绿点热力图，反映过去 35 天的 AP（行动点）获取情况。
    *   **双轨时间轴**：页面下半部分。左侧轨道渲染用户的行为（完成的任务、MacroDroid 传来的玩手机记录）；右侧轨道渲染 AI 的行为（AI 完成了它的日程、AI 给你写了日记）。两条时间线在视觉上平行推进，体现羁绊。
    *   **AI 日程面板**：展示 `owner: 'ai'` 的任务，让用户知道 AI 同桌今天要干什么。
    

### Tab 4: 角色面板 (Tab4_RPG.vue)
*   **视觉风格**：**现代轻复古 (Modern Soft Fantasy)**。参考星露谷物语的排版逻辑，但**绝对禁止使用粗黑线条和像素 Emoji**。使用低饱和度莫兰迪色系、圆角卡片和精致的单色 SVG 图标。支持页面上下滑动。
*   **核心区块**：
    *   **双人羁绊区**：顶部左侧为用户自定义头像，右侧为 AI 立绘。中间连接着“羁绊等级 (Level)”和经验条。
    *   **技能雷达图**：根据用户完成的不同 Category 任务，动态生成六维雷达图（执行力、专注、学术等）。
    *   **九宫格 Bingo (核心玩法)**：3x3 的网格组件。显示当前拥有的 AP（完成任务获得）。消耗 AP 点亮格子，前端算法检测横、竖、对角线连线。连成一线触发全屏特效，奖励海量 EXP，并触发 AI 专属夸奖。
    *   **本周 Boss**：展示一个虚拟 Boss 卡片及血条，用户和 AI 每天完成任务都会扣除 Boss 血量。

### Tab 5: 记忆与灵感 (Tab5_Diary.vue)
*   **视觉风格**：私密手账本风格。
*   **功能逻辑**：
    *   顶部提供 Toggle 切换：`[ 每日手账 ]` 与 `[ 灵感碎片箱 ]`。
    *   **手账模式**：用户可建立文件夹写日记，点击“纸飞机”图标可将日记分享给 AI，AI 会在全局聊天抽屉中给出长篇回复。
    *   **灵感碎片箱 (Zettelkasten)**：一个巨大的快速输入框，回车即存入 `db.idea_fragments`。下方列出未处理的碎片。
    *   **魔法按钮**：底部放置一个显眼的 `[ ✨ 召唤导师：连点成线 ]` 按钮。点击后，将所有未处理碎片打包发给大模型，要求 AI 找出内在联系，生成一个学术 Idea 或文章大纲，并清空碎片池。
